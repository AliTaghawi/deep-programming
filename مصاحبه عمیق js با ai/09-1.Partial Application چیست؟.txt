# Partial Application چیست؟

Partial Application تکنیکی است که با آن می‌توان بخشی از آرگومان‌های یک تابع را «پیش‌پر» کرد و تابع جدیدی ساخت که بقیهٔ آرگومان‌ها را دریافت کند.

---

## ۱. تعریف ساده

- شما تابع اصلی را دارید:  
  ```javascript
  function multiply(a, b, c) {
    return a * b * c;
  }
  ```
- با Partial Application‌ می‌گویید: «بیایید مقدار `a` و `b` را فیکس کنیم و تابع جدیدی بسازیم که فقط `c` را بگیرد.»

---

## ۲. مثال با `bind`

```javascript
// متد bind آرگومان‌های ابتدایی را قفل می‌کند و یک تابع جدید برمی‌گرداند
const doubleAndTriple = multiply.bind(null, 2, 3);
// حالا فقط نیاز داریم مقدار c را بدهیم:
console.log(doubleAndTriple(4)); // معادل multiply(2,3,4) → 24
```

- ورودی اول `bind` مقدار this را می‌گیرد (برای توابع عادی اغلب `null` می‌دهیم)  
- ورودی‌های بعدی آرگومان‌هایی هستند که می‌خواهیم از قبل ست شوند

---

## ۳. مثال با Closure (بدون bind)

```javascript
function partial(fn, ...presetArgs) {
  return function(...laterArgs) {
    return fn(...presetArgs, ...laterArgs);
  };
}

const multiplyBySix = partial(multiply, 2, 3);
console.log(multiplyBySix(5)); // 30
```

- `partial` آرگومان‌های پیشین را در closure نگه می‌دارد  
- تابع برگشتی آرگومان‌های جدید را می‌پذیرد و همه را به تابع اصلی می‌دهد

---

## ۴. مزایا و کاربردها

- ساده‌تر کردن فراخوانیِ توابعی با پارامترهای ثابت  
- خواناتر کردن کد وقتی بارها بخشی از آرگومان‌ها یکسان است  
- ساخت helper‌های تخصصی (مانند `logInfo = log.bind(null, 'info')`)

---

## ۵. تفاوت با Currying

| ویژگی               | Partial Application               | Currying                           |
|----------------------|-----------------------------------|------------------------------------|
| هدف                 | فیکس کردن برخی آرگومان‌ها        | تبدیل تابع چند‌آرگومانی به زنجیرهٔ توابع یک‌آرگومانی |
| خروجی               | تابعی با «بقیه»ٔ پارامترها       | توابعی که هر بار دقیقا یک پارامتر می‌گیرند |

مثال Currying:
```javascript
const curryMultiply = a => b => c => a * b * c;
curryMultiply(2)(3)(4); // 24
```

---

با این روش‌ها می‌توانی کدهای منعطف‌تر و خواناتری بنویسی که بخش‌های ثابت پارامترها را از اجرای اصلی جدا می‌کنند. اگر دوست داری مثال‌های بیشتری ببینی یا در پروژه واقعی پیاده‌سازی کنیم، بگو تا جلوتر بریم!
